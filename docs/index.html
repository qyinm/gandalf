By design, Hem can inspect your setup without executing MCP commands, scripts, or plugins. It never phones home. It never alters your configuration without explicit consent. It stores metadata-first and omits raw secrets. Your agent setup should be safe to experiment with and easy to restore.

Local Setup History

See what is installed, save restore points, compare changes, and recover the setup that worked.

Save Setup

Capture your current agent setup — prompts, MCP servers, skills, hooks, permissions, and settings — as a local restore point.

Compare

Compare any saved setup with your current setup. See MCP additions, skill removals, prompt edits, permission shifts, and env key changes.

Setup Notes

Surface useful restore notes before applying changes: local commands, remote MCP URLs, required env keys, and manual setup gaps.

Inventory

See what each agent has installed right now: skills, MCP servers, hooks, permissions, project instructions, and config files.

Restore

Preview exactly what will change, save the current setup first, and restore the previous agent setup when experiments break.

Profiles & Bundles

Keep named local profiles and export them as portable .hem bundles for another Mac or a clean reinstall.

Install Hem

$ npm install -g @qxinm/hem
$ hem scan --project .

Requires Node.js 22 or later

How to Use

Start with local history. Add cloud profiles later when you need multi-machine sync.

01

Inspect

Scan your project to detect agents, MCP servers, skills, hooks, env keys, and settings.

hem scan --project . --explain
02

Save

Create a baseline snapshot before you let agents install MCPs, skills, or hooks.

hem snapshot create --name baseline --metadata-only --project .
03

Compare

After changes, compare against baseline and inspect where each changed setting came from.

hem diff baseline current --project .
hem provenance current --project .
04

Restore

Generate a dry-run plan, then restore only after you have seen the exact changes. Apply mode requires --experimental.

hem restore --snapshot baseline --dry-run --project .
hem restore --snapshot baseline --apply --experimental --project .
05

Move

Export a saved setup to a portable .hem bundle. Preview Mac readiness before applying content.

hem bundle export --name baseline --out backup.hem --project .
hem bundle inspect backup.hem
hem doctor --project .
hem bundle import backup.hem --dry-run --project .

Hem starts as a local tool. Your setup history stays on your machine unless you choose to export or sync it.
